How do I install email templates?

For Magento 1.9.x.x

Before you make any changes, it is important to backup your data.
  1. Unzip the file.
  2. Open the folder “magento1”.
  3. Copy “email” folder to 
    app/locale/[langCode]/template/
  4. Copy “email-inline.css” file to 
    skin/frontend/[YOURPACKAGE]/[YOURTHEME]/css/email-inline.css
  5. Copy “email-non-inline.css” file to 
    skin/frontend/[YOURPACKAGE]/[YOURTHEME]/css/email-non-inline.css
  6. Make sure to flush all caches: System > Cache Management
TIP: Use your language Code: “en_US” is for English (United States), “de_DE” for German (Germany) etc.

How do I customize the email templates?

Customize email templates in the filesystem


Customize header and footer templates
Every frontend email template includes a header and footer template using these two directives:
{{template config_path="design/email/header"}}
{{template config_path="design/email/footer"}}
By default, those two directives load contents from these files: 
app/locale/[langCode]/template/email/html/header.html
app/locale/[langCode]/template/email/html/footer.html

You can customize header and footer templates using your favorite IDE or text editor.

You can add a logo to emails by uploading it in the Magento Admin
  • Navigate to SYSTEM > Configuration > General > Design > Transactional Emails > Logo Image
  • Upload your logo
  • Enter values for Logo Width and Logo Height
  • Click the Save Config button
Inline & Non-inline styles
Inline & Non-inline styles come from the file 
skin/frontend/[YOURPACKAGE]/[YOURTHEME]/css/email-inline.css
skin/frontend/[YOURPACKAGE]/[YOURTHEME]/css/email-non-inline.css
In the case if you do not use functions in the admin area of magento to customize email templates, you do not need attach emails in the system. Magento will use the new email templates as default email templates. This method is recommended.
Localizing Transactional Emails
To use email templates for other language than en_US, you have to replace files in english language with standard translated files (Language Pack) in the folder: 
app/locale/[langCode]/template/email/
TIP: Except the html folder.

Customize email templates using the Magento Admin

To customize a default Magento email template, simply navigate to the Transactional Emails page in your Magento backend via the path SYSTEM > TRANSACTIONAL EMAILS, then follow these 3 easy steps:

  1. 1. click ADD NEW TEMPLATE to create a new custom email template
  2. 2. select the default transactional email template that you wish to customize from the TEMPLATE DROP-DOWN
  3. 3. select your LANGUAGE from language drop-down
  4. 4. click LOAD TEMPLATE
  5. 5. you will see that the HTML code for that email now appears in the TEMPLATE CONTENT section below


  7. 6. now you can EDIT HTML code of the email template
  8. 7. after editing you can SAVE the edited TEMPLATE with a custom name
Now that you have created your custom transactional emails in Magento, it is time to configure your system configuration panel. This will tell Magento to send your new custom email templates (instead of the default templates) when that particular action is triggered on your site.
Configure the New Order Email
To configure the newly customer New Order email, Navigate to the SYSTEM > CONFIGURATION panel, select SALES EMAILS from the sales tab. Click on the ORDER row and select your custom template from the NEW ORDER EMAIL TEMPLATE dropdown:


Configure the New Shipment Email
To configure the newly customer New Shipment email, simply navigate to the SYSTEM > CONFIGURATION panel, and select SALES EMAILS from the sales tab. Here you will click on the SHIPMENTS row and select your custom template from the SHIPMENT EMAIL TEMPLATES dropdown:


Configure the Forgotten Password Email
To configure the Forgot Password email, simply navigate to the SYSTEM > CONFIGURATION panel, and select CUSTOMER CONFIGURATION from the customers tab. Here you will click on the PASSWORD OPTIONS row and select your custom template from the FORGOT PASSWORD TEMPLATES dropdown:
